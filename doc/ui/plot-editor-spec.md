# ◇ 漫画プロットエディタ — 仕様書

## コンセプト

**「感情設計 → 演技メモ → ページ割り → コマ割り」までをつなぐ漫画ネーム前エディタ。**

既存ツール（Scrivener / Plottr / CLIP STUDIO）は「書く」か「描く」に寄っている。
このツールはその間にある **「話をページに落とす」フェーズ** を専用UIで支援する。

### ターゲットユーザー

- 漫画・同人誌のプロット〜ネーム前段階を整理したい人
- コマ割りが億劫で、テンプレートから選びたい人
- 感情設計を軸にシーンを組み立てたい人

---

## 現在の実装状況（v0.1）

### データ階層

```
作品（タイトル、テーマ、メモ）
  └ 場面カード（タイトル、プロット、何を伝えたいか）
       └ ページ（ページ番号、コマ割りテンプレート、見せゴマ）
            └ コマ（内容・構図、感情・行動原理）
                 └ セリフ（話者、セリフ内容）※複数可
```

### 実装済み機能

#### 場面カード
- 縦タイムライン表示（ドラッグで並べ替え可能）
- 折りたたみ式（閉じた状態でページ数・コマ数のサマリー表示）
- プロット欄（この場面で起きること）
- 「何を伝えたいか」欄（シーンの本質・読者に残したい感情）
- 場面の追加・削除（削除時はカスタム確認ポップアップ）

#### ページ
- ページごとにコマ割りテンプレートを選択可能
- テンプレートは12種類（全面、縦2分割、横2段、横3段、上2+下1、上1+下2、4コマ格子、L字右大/左大、5コマ変則、上ワイド+下3、段差3コマ）
- コマ割り読み順は **右→左**（漫画の読み方に準拠）
- テンプレート選択時、現在のコマ数に合うものを優先表示（他のコマ数は「もっと見る」トグル）
- テンプレート変更時にコマ数が自動調整される（増減）
- ページのビジュアルマップ表示（コマの配置・内容・話者がプレビューで見える）
- ページの追加・削除

#### 見せゴマ
- 各ページで1つのコマを「見せゴマ」に指定可能（任意。指定しなくてもOK）
- 新規ページ作成時はデフォルトで最初のコマが見せゴマに設定される
- コマカードの空白部分をタップで見せゴマに切り替え（入力欄タップ時は編集が優先）
- **見せゴマを再タップで解除可能**（全コマ均等な見た目に戻る）
- 「見せゴマ」テキストラベルで明示（⭐アイコンは「お気に入り？」と誤解されるため廃止）
- 見せゴマは金色の枠・背景で視覚的に区別
- コマ割りマップ上でも金色ハイライト＋「見せゴマ」テキスト表示
- 1コマのみのページでは見せゴマUI非表示（選ぶ必要がないため）
- ※見せゴマ＝大きいコマとは限らない。あくまで「このページで一番伝えたいコマ」の印。サイズはコマ割りテンプレートで別途制御する設計

#### コマ
- コマ内容・構図の入力
- 感情・行動原理フィールド（♡マーク、薄紫の背景で視覚的に区別）
  - 「なぜこの演技？」を書く欄。感情設計→演技への橋渡し
- コマの追加・削除

#### セリフ
- 1コマに複数セリフ対応（交互の会話、掛け合い）
- セリフごとに話者（ドロップダウン）とセリフ内容を入力
- 「+ セリフ追加」で何行でも追加可能
- セリフの個別削除

#### 登場人物管理
- サイドバーで登場人物の追加・削除
- 登録した人物がセリフの話者ドロップダウンに反映

#### 参考コマ割り
- サイドバーの「参考コマ割り」タブで管理
- 好きな作品のコマ割りパターンを名前・テンプレート・メモ付きで保存
- ページのコマ割り選択時の参考として閲覧

#### 集計
- 総ページ数・総コマ数・場面数をヘッダーに常時表示
- サイドバーに詳細集計（場面数、総ページ数、総コマ数）
- キャラ別セリフ数のカウント

#### データ管理
- JSON形式でのエクスポート（全文コピー → メモアプリに貼り付け保存）
- JSON形式でのインポート（貼り付け → 復元）
- データリセット（カスタム確認ポップアップ付き）
- HTML版ではlocalStorageによる自動保存対応

### UI / UX上の決定事項

| 決定 | 理由 |
|------|------|
| 縦タイムライン | 漫画の流れが上→下で自然。スマホでスクロール操作と親和性高い |
| 右→左の読み順 | 漫画の読み方に準拠。コマ番号が右上→左→下段右→左の順 |
| 折りたたみ式カード | スマホで全場面を一覧しつつ、必要な場面だけ展開 |
| 見せゴマはテキストラベル | ⭐アイコンだと「お気に入り？」「機能がある？」と誤解される。「見せゴマ」と書く方が目的が伝わる |
| 見せゴマは任意（解除可能） | 見せゴマ＝大きいコマとは限らず、そもそも見せゴマを設定しないページもありうる。必須にすると実態に合わない |
| 見せゴマのopacity制御は不採用 | 非見せゴマが薄くなるとコマ内容が読みにくい。ラベル＋金色枠だけで十分 |
| 見せゴマのscale拡大は不採用 | 固定テンプレートの絶対配置でscaleすると表示が崩れる。将来CSS Grid（fr単位）で比率制御すれば実現可能 |
| カードタップで見せゴマ切替 | ⭐ボタン単体はスマホで小さすぎる。カード空白部タップの方が直感的 |
| 削除は全てカスタムポップアップ | `confirm()` がアーティファクト環境で動かない。カスタムモーダルで統一 |
| セリフは配列構造 | 1コマ内で複数キャラが喋る漫画表現に対応 |
| 感情欄は薄紫 | プロット（白）、伝えたいこと（薄橙）、感情（薄紫）で視覚的に役割を分離 |

## 技術仕様

### 環境

- **アーティファクト版**: React (JSX)、Claude.aiのアーティファクト環境で動作
  - `window.storage` APIは環境によって使えない場合あり
  - JSON コピペ方式でデータ保存
- **HTML版**: React (CDN)、単体HTMLファイル
  - `localStorage` で自動保存
  - スマホのホーム画面に追加でアプリ化可能

## デザイン方針（v0.2 クールトーン・リニューアル）

v0.1 はテラコッタ基調の暖色だったが、Web アプリ化（v0.2）に合わせて
**寒色ベースのクリーン・知的な方向**へ刷新する。
Notion / Linear 系のトーンで、長時間のテキスト入力に疲れない画面を目指す。

検討の経緯と比較は `doc/ui/cool-palette-comparison.jsx`（寒色3パレット × 見せゴマカラー）を参照。

### 基本トーン

寒色・低彩度・高明度の背景。アクセントは青紫（インディゴ）で知的・モダンな印象。
彩度の高い色は「見せゴマ」「感情」「削除」など意味のある箇所にだけ使い、画面全体は静かに保つ。

### パレット：インディゴ

| 用途 | カラー |
|------|--------|
| 背景 | `#F8F8FA` |
| カード | `#FFFFFF` |
| カード枠 | `#EBEBF0` |
| アクセント | `#5B6AD0`（インディゴ） |
| アクセント淡 | `#EDEFFF` |
| テキスト | `#1A1A2E` |
| テキスト副 | `#9090A0` |
| 伝えたいこと | `#F6F3FF` / `#E4DEFF` |
| 感情・行動原理 | `#FFF5F9` / `#FFE0EC` / `#C0608A`（テキスト） |

3パレット（インディゴ / スレートブルー / ミント）を比較したうえで、
**一番モダンで万人受けするインディゴを採用**。スレートブルーは無骨すぎ、ミントは好みが分かれるため見送り。

### 見せゴマカラー：ティール / アンバー

見せゴマ（このページで一番伝えたいコマ）は、ベースのインディゴと色相をずらして際立たせる。
インディゴに合う2案を用意し、作品の雰囲気で選べるようにする。

| 案 | 枠・アイコン | 背景 | 枠（淡） | 性格 |
|----|-------------|------|---------|------|
| ティール | `#1A9A8A` | `#F0FDFB` | `#80D8CC` | 静か・爽やか。青系とよく馴染む |
| アンバー | `#B08828` | `#FFFBF2` | `#E0C878` | 暖かい・目を引く。v0.1 金色の系譜 |

※ 見せゴマ ≠ 大きいコマ。色は「印」であり、サイズはコマ割りテンプレートで別途制御する（v0.1 の決定を継承）。

### アイコン：Lucide

絵文字（⭐💬♡）や場当たりの SVG をやめ、**`lucide-react` で統一**する。
- 線の太さは `strokeWidth={2}`、サイズはコンテキストで 11〜18px を基準に揃える
- 例：感情 = `Heart`、セリフ = `MessageCircle`
- 色はその場のテキスト色／アクセント色に合わせ、アイコン単体で主張させない

### ガラスモーフィズム

オーバーレイ層（ヘッダー、サイドバー、モーダル）に半透明 + `backdrop-blur` を適用し、
背面のコンテンツがうっすら透ける奥行きを出す。
- 本文カード（場面・ページ・コマ）は不透明のまま。読みやすさを最優先するため、ガラス効果は使わない
- ブラーは控えめに。やりすぎると重く・安っぽくなる

### マイクロモーション

操作に対する軽い動きで「反応している」感触を出す。やりすぎず、機能の邪魔をしない範囲。
- トランジションは `0.15s` 前後を基準
- 対象：タブ切替、カード展開／折りたたみ、見せゴマ切替、ホバー、モーダル開閉
- アニメーションは色・影・透明度・位置の微小変化にとどめ、派手な演出はしない

## 将来構想

### 確定

- **Webアプリ化**: React本格実装。localStorageまたはDB連携による永続化。スマホ対応必須

### 欲しい

- **書き出し機能（クリスタEX連携）**: ページごとにセリフ・ト書きをテキスト/CSV形式で出力 → CLIP STUDIO EXのストーリーエディタに流し込む想定。PDF出力は優先度低（印刷/共有用途）
- **ページめくりプレビュー**: 全ページを見開き/単ページで流し読みできるビュー。課金機能にするか常設にするかは要検討

### いつか欲しい

- **ドラッグでコマ境界線を調整**: react-resizable等でFigmaライクにコマサイズを自由に変更
- **カスタムコマ割り作成**: テンプレートに加えて、自分でコマ割りを描ける機能
- **参考画像の貼り付け**: コマ割り参考として画像を保存・閲覧

### 保留（必要性を検証中）

- **CSS Gridでコマ割り比率制御**: `fr` 単位で見せゴマを大きく表示する等。本当に欲しいかまだわかっていない
- **ネームラフ描画**: コマ内にシンプルな線画を描けるキャンバス
- **感情グラフ**: 場面ごとの感情の起伏をグラフ化

### 不採用

| 機能 | 理由 |
|------|------|
| BL特化フィールド | 汎用ツールとして設計。ジャンル特化は不要 |
| キャラ別タイムライン | 短編〜中編の同人誌では使いどころが薄い |
| BOOTH販売 | 販売は想定しない。Webアプリとして自分で使う・公開する方向 |
