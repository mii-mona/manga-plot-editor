## 設計の前提（このアプリの立ち位置）

| | 本アプリ | nameboard（参考） |
|---|---|---|
| 軸 | **感情設計→演技→ページ→コマ** の物語パイプライン | 見開きの実コマ割り（ネーム盤面） |

## P1 — エディタの基礎体力（最優先）

### 1. Undo / Redo
- **なぜ**: 編集ツール最重要級の欠落。誤削除が即 localStorage に保存されると復旧不能。参考ツールにはあり、本アプリには無い。
- **方針**:
  - `PlotData` 全体のスナップショットを履歴スタックに積む方式が実装が単純（状態が大きくないため許容）。
  - `useAutoSave` の自動保存が履歴を汚さないよう、history push は「ユーザー操作起点の変更」に限定する。
  - 操作: PC は `Ctrl+Z` / `Ctrl+Shift+Z`、スマホはヘッダーに ↩︎ / ↪︎ ボタン。
  - スタック上限（例: 50）でメモリを抑える。
- **規模**: 中。状態更新が `App.tsx` に集約されているか、reducer 化が必要か先に確認。

### 2. データ消失対策（バックアップ強化）
- **なぜ**: 現状 localStorage のみ。ブラウザのデータ削除で全消失。`manga-project-data` への手動コミットは摩擦が大きい。
- **方針**:
  - **ワンクリック JSON ダウンロード**（`Blob` + `a.download`）。既存の「書き出し（コピペ）」に追加。
  - **File System Access API** で「ファイルに保存／上書き保存」（対応ブラウザのみ。非対応はダウンロードにフォールバック）。
  - 最終バックアップ日時の表示、または一定操作ごとにエクスポートを促す軽い通知。
  - `manga-project-data` リポジトリ運用（`project-x/latest.json` + `backups/`）への手動コミットと自然につながる導線。
- **規模**: 小〜中。

## P2 — 制作フロー強化（spec の「欲しい」＋参考でも有効性が裏付け）

### 3. テキスト / CSV 書き出し（クリスタEX連携）
- **なぜ**: ネームを実際に描く工程への"出口"。出力したらすぐ CLIP STUDIO に流して描き始められる、制作フローの次工程を直接アンロックする機能。規模が小〜中で効果/労力が高い。参考ツールの汎用 export より目的が尖っており差別化にもなる。
- **方針**:
  - ページごとにセリフ・ト書きをテキスト/CSV で出力 → CLIP STUDIO EX ストーリーエディタに流し込む想定。
  - PDF 出力は優先度低（印刷/共有用途）。
- **規模**: 小〜中。

### 4. 見開き / 読み通しプレビュー
- **なぜ**: 参考ツールは見開き単位が主役。本アプリの spec「ページめくりプレビュー」を実装すると、**右ページ＝めくりの引き**といった"見せ場の配置確認"ができ、既存の見せゴマ機能と相性が良い。"魅せ/レビュー"寄りで、無くても制作は進められるため書き出しより後。
- **方針**:
  - 全ページを見開き（右←左）で流し読みするビュー。単ページ表示も切替可能。
  - 見せゴマのハイライトをプレビュー上でも表示。
  - 課金 or 常設は未定（spec 記載）→ 当面は常設で実装、判断は後回し。
- **規模**: 中。

---

## P3 — 個人ツール / スマホ運用フィット

### 5. PWA 化（インストール可能・オフライン）
- **なぜ**: 「スマホにインストールして使う自分用ツール」が完成する。localStorage ベースと噛み合う。[index.html](../index.html) に既に `apple-mobile-web-app` メタがあり半分準備済み。
- **方針**:
  - `manifest.json`（アイコン・テーマカラー `#C2785C`・display standalone）。
  - service worker（`vite-plugin-pwa` 推奨）でアプリシェルをオフラインキャッシュ。
  - `public/data/sample.json` のキャッシュ戦略に注意（更新時に古いキャッシュを掴まないように）。
- **規模**: 小〜中。

## 当面やらない（投資対効果が低い）

| 項目 | 理由 |
|------|------|
| 多言語対応（KO/ZH 等） | 自分用＆日本語前提。国際公開を狙う段階になれば再検討 |
| 盤面の自由コマ移動（Figma 風） | 重い。ドラッグ境界調整（保留枠）で段階的に検証してから |

---

## 推奨着手順

1. **Undo/Redo**（基礎・安全性）
2. **データ消失対策**（DL / ファイル保存）
3. **テキスト/CSV 書き出し**（描く工程への出口・効果/労力が高い）
4. 見開きプレビュー（魅せ・レビュー）
5. PWA 化

> 基礎（安全に編集できる）→ 制作フロー（描く工程へつなぐ）→ 体験（インストール）の順。
